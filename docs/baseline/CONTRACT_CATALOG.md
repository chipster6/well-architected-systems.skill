# Contract Catalog (Baseline Stub)

## Purpose
List the boundary contracts the system is expected to expose or consume. Full contract specs and schemas are authored by `arch-docs`.

## Contract families (initial)
- External API contracts: request/response contracts for public or partner APIs.
- Event contracts: event payload schemas for any pub/sub or streaming interface.
- Data contracts: shared data shape agreements between services and stores.

## Current status
No concrete contracts are defined in baseline. When introduced, each contract must include:
- stable contract ID
- owner and consumer(s)
- versioning policy
- schema + at least one example
